   BLIBG ＜汎用＞
    旧版日立開発環境評価版用ライブラリアン


1. 概要

古い日立開発環境評価版のためのライブラリアンです。HOSのカーネルライブ
ラリを、ライブラリアンを含まない旧版の開発環境評価版で作成するために製
作されました。完全にライブラリファイルの仕様を把握している訳ではないの
で、その動作・機能について一切の保証はありません。利用は自己責任でお願
いします。


2. 使い方

1)  BLIBG libraryname objfile1 [ objfile2 ... ]
    libraryfile: 出力ライブラリファイル名
    objfileN   : 入力オブジェクトファイル名
    (N=1,2,...)

    コマンドラインから、出力するライブラリファイル名と入力するオブジェ
    クトファイル名を与えます。ライブラリファイル名と一つ以上の入力オブ
    ジェクトの指定が必要です。


2)  BLIBG -sub=subcmdfile
    subcmdfile: ライブラリアンサブコマンドファイル名

    出力ライブラリファイル名、入力オブジェクトファイル名を、ライブラリ
    アンサブコマンドファイルを使って指定します。サブコマンドファイルは
    テキストファイルで、出力ファイル名指定のoutput、入力ファイル名指定
    のinputの記述が出来ます。

    output libraryfile
    input  objectfile

    チェックの類は殆んどありません。少なくとも一つ以上のinput指定、
    output指定が必要です。output指定が複数の場合、最後のものが有効にな
    ります。


3. その他

最大256(標準)のオブジェクトファイルを指定出来ることになっていますが、
オブジェクトファイル中に含まれるシンボル情報量や、サブコマンドファイル
を使っている場合はファイル名情報の量によって、その数は保証されません。
リンクするオブジェクトファイル中で広域定義可能なシンボル数は、オブジェ
クトファイル当たり最大64(標準)です。

オブジェクトファイルの厳密なチェック等はしていないため、H8とSHのオブジェ
クトが混在したライブラリも出来てしまいます。

BLIBGが生成するライブラリファイル中の全ての日付時刻情報は、入力オブジェ
クトファイル、操作時刻等によらず、常に 020916113234(標準)になっていま
す。しかし、特にこれが問題になることはないようです。ちなみに、この日付
時刻に深い意味はありません。


もしBLIBGが受け付けないオブジェクトファイルに遭遇したら、それを
Project HOSに報告していただければ、対策が取れる場合があるかもしれませ
ん。その際には、同梱されているOBJCHKを使ってそのファイルをチェックして
下さい。

OBJCHK filename > report.txt

報告に必要なのはこうして得られたreport.txtの、データでない印字可能文
字列(大域定義シンボル、ファイルタイムスタンプ、アセンブル・コンパイル
タイムスタンプ、ツール識別子等)を含んだブロックと思われる部分だけでお
そらく十分です。他のブロックは削除してしまって下さい。

上の文は、対策をとることを保証、約束する類の意味を一切含んでいないこと
をここに確認しておきます。
